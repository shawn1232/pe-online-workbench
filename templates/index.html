AI + Agent 系统定位与五大模块

这不是“AI 帮我看项目”的玩具，而是一套把一级PE研究流程化、证据化、可解释化的能力框架。

1

信息采集与赛道映射

围绕政策、行业协会、展会发布、公司官网、融资事件、客户验证、招投标、年报/招股书等公开信息进行持续采集，并映射至既有赛道库，识别其属于哪个细分赛道、对应哪条投资主线，以及属于需求催化、技术突破、客户验证还是资本事件。

2

未上市候选公司池构建

以未上市公司为核心筛选对象，优先纳入处于B轮前后、具备一定客户验证、产品卡位清晰、业务相对聚焦、未来具备继续融资或退出可能的公司；上市公司仅作为产业参照、估值锚和退出映射。

3

横向比较与排除逻辑生成

围绕公司定位、核心产品、高端属性、客户验证、商业化成熟度、融资阶段、行业地位、退出可行性等维度，对候选公司进行横向比较，并同步生成排除逻辑。系统既回答“为什么选”，也回答“为什么不选”。

4

解释型评分引擎

针对赛道景气、利润池、高端属性、客户验证、商业化进度、可投性、退出可行性、竞争卡位和信息充分度进行结构化赋分。每一项分数都必须回溯到对应的公开证据，实现可打分、可解释、可复核。

5

最终标的推荐与动态跟踪

在候选池、横向比较和评分结果基础上，输出当前赛道重点跟踪公司、备选公司和最终推荐标的，并明确推荐理由、核心证据、待核实信息缺口，以及哪些变量变化可能导致推荐切换。

解释型评分引擎

总分不是拍脑袋给，而是由九个维度的结构化等级经过规则引擎换算得到，并叠加排除项惩罚。

总分 = Σ（九维权重 × 维度等级 / 5） − 排除项惩罚
赛道景气12分
政策催化、需求扩张、渗透率提升、产业景气度
利润池12分
价值节点位置、议价能力、毛利厚度、后市场厚度
高端属性10分
精度、可靠性、认证壁垒、工艺 know-how、替代难度
客户验证12分
标杆客户、验证周期、导入深度、复购/复制性
商业化进度10分
量产交付、订单转化、渠道与交付体系、规模化能力
可投性12分
轮次、股权结构、融资节奏、接触可能性、估值可谈性
退出可行性10分
产业买家、IPO可比、并购承接、退出路径清晰度
竞争卡位12分
差异化、行业地位、客户心智、平台化延展
信息充分度10分
公开证据完整性、交叉验证程度、信息一致性
排除项惩罚-5 到 -20
配套功能件型、工程项目型、宽口径拼装型、标准耗材型、独立客户入口弱、利润池过薄、退出锚缺失等
系统同时将九维评分自动映射为三层结果：静态判断层（赛道景气/利润池/高端属性/竞争卡位）、动态证据层（客户验证/商业化进度/信息充分度）、交易退出层（可投性/退出可行性）。

Agent / Skill 架构

不是单轮 Prompt，而是多代理协同、技能封装、规则约束、状态机推进的研究工作台。

A1
Signal Harvester · 情报采集代理多源异构公开信息接入：政策、协会、展会、官网、融资、招投标、年报/招股书、客户验证、招聘动态。
A2
Sector Mapper · 赛道映射代理对事件进行赛道识别、投资主线归类、价值节点映射，形成候选方向列表。
A3
Company Pool Builder · 候选池构建代理围绕未上市主体构建候选公司池，上市公司仅用于估值锚、产业参照与退出映射。
A4
Compare & Exclude · 横向比较代理输出“为什么选它 / 为什么不选别人 / 为什么排除”的完整横向比较与排除逻辑。
A5
Explainable Scoring Engine · 解释型评分引擎由模型输出九维等级，再由本地规则引擎做确定性换算，避免“模型直接拍总分”。
A6
Tracking State Machine · 动态跟踪代理将结果推进到“观察 / 补证 / 约访 / 深跟 / 立项 / 放弃”的状态机，并生成下一步动作建议。
